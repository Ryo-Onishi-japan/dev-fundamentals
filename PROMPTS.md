## 🔀 Git / GitHub

<!--
💡 Git と GitHub の違い
======================
- Git: バージョン管理システム（ローカルで動作）
- GitHub: Gitリポジトリをホスティングするサービス（クラウド）
gh CLI を使うと、GitHubをコマンドラインから操作できます。
-->

### リポジトリ作成・Push

<!--
💡 用語解説
==========
- git init: 新しいGitリポジトリを初期化
- commit: 変更をリポジトリに保存
- push: ローカルの変更をリモート（GitHub）に送信
- リポジトリ: コードとその履歴を保存する場所
-->

```
git initして、commitしてGitHubにghを使ってpushしてください。
リポジトリは新規作成してください。
```

<!--
↑ このプロンプトでAIに依頼すると、以下のコマンドを実行してくれます：
  1. git init （リポジトリ初期化）
  2. git add . （ファイルをステージング）
  3. git commit -m "..." （コミット作成）
  4. gh repo create （GitHub上にリポジトリ作成）
  5. git push （GitHubにpush）
-->

### ブランチ切ってPR作成

<!--
💡 用語解説
==========
- ブランチ: 開発の分岐線。mainとは別に作業できる
- Pull Request (PR): 変更をレビューして取り込む依頼
-->

```
新たにブランチを切って、commitしてpushして
ghを使ってPull Requestを作成してください。
```

### IssueをGitHubに登録

<!--
💡 用語解説
==========
- Issue: タスク、バグ、機能要望などを管理する機能
- @ファイル名: Gemini CLIでファイルを参照する記法
-->

```
@tasks.md をGitHubのissueにghコマンドで登録してください。
```

<!--
↑ @tasks.md はGemini CLIの特別な記法
  指定したファイルの内容をAIに渡すことができます
-->

## 💻 コードレビュー

<!--
💡 コードレビューとは？
=====================
他の開発者（やAI）にコードを確認してもらうことです。
バグの発見、品質向上、知識共有などの効果があります。
-->

### PRレビュー依頼

```
以下のPull Requestをレビューしてください
<PRのURL>
```

<!--
↑ PRのURLを貼り付けると、AIがPRの内容を確認してレビューします
  例: https://github.com/username/repo/pull/1
-->

### コードレビュー

```
以下のコードをレビューしてください。
改善点、パフォーマンス、セキュリティの観点で。

[コード貼り付け]
```

<!--
💡 レビューの観点
================
- 改善点: より良い書き方、可読性の向上
- パフォーマンス: 処理速度、メモリ使用量
- セキュリティ: 脆弱性、情報漏洩リスク
-->

## 📚 MCP活用

<!--
💡 MCPとは？（再掲）
==================
Model Context Protocol の略。
AIが外部サービス（API、ブラウザ、ファイルシステムなど）と
連携するための標準プロトコルです。
-->

### 最新ドキュメント参照

```
Context7を使って、[ライブラリ名]の最新の使い方を教えてください
```

<!--
↑ Context7: 最新のライブラリドキュメントを取得するMCPサービス
  例: "Context7を使って、Reactの最新の使い方を教えてください"
  AIの学習データより新しい情報を取得できます
-->

### ブラウザ操作

```
playwrightで[URL]を開いて
```

<!--
↑ Playwright: ブラウザを自動操作するツール
  MCP経由でAIがブラウザを操作できます
  例: "playwrightでhttps://example.comを開いてスクリーンショットを撮って"
-->

## 🧹 コンテキスト管理

<!--
💡 コンテキストとは？
====================
AIとの会話履歴や参照中のファイル情報のことです。
会話が長くなるとコンテキストが増え、処理が遅くなることがあります。
/clear で会話履歴をリセットできます。
-->

| コマンド | 説明 |
|----------|------|
| `/clear` | コンテキスト解放（会話履歴をリセット） |
| `/mcp` | MCPサーバー管理（接続状況の確認など） |

<!--
💡 Gemini CLIのスラッシュコマンド
================================
/ で始まるコマンドはGemini CLI内で使える特別なコマンドです。
通常のプロンプトとは異なり、CLI自体を操作します。
-->

## 💡 良いプロンプトのコツ

<!--
💡 なぜプロンプトの書き方が重要？
================================
AIは「曖昧な指示」には「曖昧な回答」を返します。
具体的で詳細な指示を出すことで、正確な回答を得られます。
-->

```
EmployeeDAOのfindByIdメソッドを作成してください。

要件:
- Oracle JDBC使用
- PreparedStatement使用
- 見つからない場合はOptional.empty()を返す

現在のEmployeeクラス:
[コード貼り付け]
```

<!--
↑ 良いプロンプトの例
  1. 何を作るか明確（EmployeeDAOのfindByIdメソッド）
  2. 要件を箇条書き（Oracle JDBC, PreparedStatement, Optional）
  3. 関連コードを添付（Employeeクラス）
-->

**ポイント:**
- 具体的な要件を箇条書き
- 関連コードを添付
- 期待する動作を明記

<!--
❌ 避けるべきプロンプト例
========================
- 「DAOクラス作って」→ どんな機能が必要かわからない
- 「完璧なコードで」→ 何をもって完璧かわからない
- 「エラーが出る」→ どんなエラーかわからない

✅ 改善例
=========
- 「findById, findAll, saveメソッドを持つEmployeeDAOを作って」
- 「本番環境で使える堅牢なエラーハンドリング付きで」
- 「NullPointerExceptionが発生します。Employeeがnullの時に起きます」
-->
